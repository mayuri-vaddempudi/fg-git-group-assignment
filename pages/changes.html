What Are Changes?

In Git, changes represent updates made to files in your repository. Each commit saves a snapshot of your project. Tracking changes helps you understand progress, collaborate with others, and maintain a clean history.

Adding code to existing repository

Cloning:

1. Adding code to your repository starts with cloning that repository. Cloning a repository creates a local copy that you can use to do your work. By keeping this local copy synchronized with the remote repository stored in GitHub, you're able to get the latest changes and push your updates so others can receive them.

  • git clone repository-url
  • cd repository-name

Staging, Committing & Pushing Changes

Staging:

1. Staging in Git involves adding new, modified, or deleted files to a staging area before committing them. This allows for flexibility in choosing the files to commit.

2. Before staging add new files or Edit existing files in your local working directory.

3. Check the status of your repo.This shows which files have been changed, added, or deleted.

  • git status
  • git add . (stages all files)
  • git add filename (stage a file)

Committing:

1. Commit the staged files with a meaningful message.

2. Check the status again by running this command: git status

3. If all changes have been committed, and there are no untracked files, it should say: nothing to commit, working tree clean.

  • git commit -m "Comment"

Pushing:

1. Pushing meanes sending your commits to the remote repository (ex:GitHub).

2. If the tracking information has already been set (when you did your first push, or if you originally cloned the repo from a remote server) you can use a simpler push command.

3. Run either of these commands:

  • git push
  • git push origin main
