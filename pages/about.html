About Our Project

Our team consists of dedicated students working collaboratively on a web development project. Each member brings unique skills and perspectives that contribute to the project's success. Through effective communication and division of responsibilities, we have created a cohesive working environment where everyone's contributions are valued.

Working in a group project requires coordination, mutual respect, and clear role definitions. Our team holds regular meetings to discuss progress, resolve issues, and ensure that all components integrate smoothly. We use version control systems and collaborative tools to maintain code consistency and track changes effectively.

How We Work Together

Collaboration is key to our project's success. We follow a structured workflow that includes planning, task division, and regular check-ins to ensure everyone is aligned and progressing towards our common goals.

1. Planning & Communication

Our project begins with a planning phase where we define objectives, set timelines, and assign tasks. We use messaging platforms and video calls for daily standups and weekly progress reviews. Our team members report their progress and any blockers they face, ensuring transparency and accountability.

2. Work Distribution

Tasks are distributed based on individual strengths and expertise. Mayuri Team leads the frontend design and ensures all members understand their roles. Anuhya handles project structure of architecture while collaborating with frontend developers to ensure smooth integration. Suneetha focuses on implementing designs into functional code. Sowjanya creates content and tests features to identify bugs early.

3. Code Integration & Version Control

We use Git for version control, allowing multiple team members to work simultaneously without conflicts. Each member works on their own branch and submits pull requests for review before merging. This practice ensures code quality and prevents breaking changes from affecting the main project.
