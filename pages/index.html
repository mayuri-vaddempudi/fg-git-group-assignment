Welcome to Our Git Project

A collaborative journey into Git & GitHub workflows

In Git projects, we can track changes, work with branches, collaborate with pull requests, resolve conflicts, and contribute to open source.

What is Git?

Git is a version control system that helps developers track changes in their code, collaborate with teammates, and manage different versions of a project. It allows multiple people to work on the same project without overwriting each other's changes.

Key points about Git:

Version control → Records changes to files over time so you can revert, compare, or restore previous versions.

Commits → Each change is saved as a snapshot in history.

Branches → Lets you create separate “lines of development” for features, fixes, or experiments.

Merging → Combines changes from different branches back together.

Distributed → Every developer has a full copy of the repository, not just the latest version.

Offline use → You can commit, branch, and explore history without internet.

What is GitHub?

GitHub is a platform for version control and collaboration. It helps developers track changes in code using Git and makes it easier to share, review, and work together on projects from anywhere.

Key points about GitHub:

Remote Repositories → your code is backed up and accessible anywhere.

Collaboration Tools → Pull Requests (PRs), code reviews, and inline comments.

Project Management → Issues, labels, discussions, and project boards.

Automation → GitHub Actions for CI/CD (automated builds, tests, deployments).

Community & Open Source → Share your repositories publicly and contribute to others.

About This Project

This project is a collaborative group assignment designed to help us learn and practice Git and GitHub workflows. Our main goal is to understand how teams can work together on the same codebase while keeping everything organized and conflict-free.

  • Create and manage branches for different features
  • Work with pull requests to review and merge code
  • Learn how to resolve merge conflicts
  • Protect important branches like main
  • Document contributions and teamwork clearly

The final output is a multi-page website where each page demonstrates different aspects of version control — branches, changes, and project history.
